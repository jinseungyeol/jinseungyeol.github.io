# 세모리포트

> 퍼블리싱 · 2024.01 ~ 2024.12 · 참여 100% · HTML, CSS, jQuery · 하이브리드 앱

세무 리포트 서비스 '세모리포트' **하이브리드 앱**의 퍼블리싱을 전담한 프로젝트입니다. 공통 컴포넌트를 정의한 마크업 가이드(공통 UI 컴포넌트 가이드)를 먼저 구축한 뒤 이를 조립해 60여 개 화면을 일관된 구조로 생산했고, 이 방식으로 **개발 속도 향상과 효율성 개선**이라는 성과를 만들었습니다.

## 프로젝트 배경

하이브리드 앱은 WebView 렌더링 특성상 다양한 해상도에서 화면이 깨지지 않아야 했고, 화면 수가 많아 **컴포넌트 재사용 없이는 일정을 맞추기 어려운 구조**였습니다. 이에 화면을 바로 찍어내는 대신 마크업 가이드를 선행 구축해, 이후 화면 작업이 "가이드에서 컴포넌트를 골라 조립"하는 방식이 되도록 판을 깔았습니다.

## 주요 작업

| 영역 | 내용 |
|---|---|
| 마크업 가이드 구축 | 공통 컴포넌트·화면 가이드를 정의해 개발 리소스 절약, 일관된 UX 제공 |
| 앱 화면 퍼블리싱 | 리포트·정산·세금·계좌·인증서·설정·알림 등 도메인별 60여 개 화면 마크업 |
| 산출물 관리 | 화면 채번 규칙과 워크리스트 페이지로 기획·개발 파트와의 전달 체계 정비 |

## 구현 포인트

- **가이드 문서화** — 각 컴포넌트 페이지에 마크업 기준 표(구조·cssClass명·연결 JS 함수), 예시 코드, 360×640 iframe 목업 프리뷰를 함께 배치해 개발자가 복사·조립만으로 화면을 만들 수 있게 했습니다.
- **컴포넌트 체계** — Base(기본구조·폰트·헤더·본문 레이아웃·슬라이드팝업·센터팝업·여백)와 Modulers(리스트 상세형·선택형·편집형·이동형)로 계층을 나누고, 확인 메시지·하단 고정 버튼 등 패턴별 목업을 30여 종 제공했습니다.
- **화면 채번 규칙** — `1Depth_2Depth_일련번호(+A~Z)` 네이밍 컨벤션을 정의해 분기 화면과 팝업까지 산출물을 체계적으로 관리했습니다.
- **WebView 대응** — 주요 리포트 화면의 반응형(RE) 버전을 별도 구축하고, Pretendard 웹폰트·Swiper·mobiscroll(날짜 선택) 등을 조합해 앱 WebView에서 안정적으로 렌더링되도록 했습니다.

## 데모 구성

- **App 데모** — 도메인 탭(사용자·알림·리포트·정산·세금·설정 등)과 검색 필터가 있는 산출물 목록에서 각 화면을 열어볼 수 있습니다. 회원 가입, 계좌·인증서 관리, 리포트 조회, 급여·세금 화면 등 실제 앱 화면 마크업이 포함되어 있습니다.
- **UI 가이드 데모** — 상단/좌측 메뉴/콘텐츠 iframe 구조의 가이드 사이트로, 컴포넌트별 기준 표와 목업 프리뷰를 확인할 수 있습니다.
