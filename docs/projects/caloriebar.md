# 칼로리바

> 개발 · 참여 65% · HTML, CSS, JavaScript, Django, MySQL

다이어트·체형관리 프랜차이즈 칼로리바의 **고객 홈페이지 + 지점 백오피스**. TV홈쇼핑·온라인 광고로 유입된 체험 예약이 수십 개 지점(직영+가맹)으로 나뉘어 들어오는 구조라, 고객 랜딩부터 본사·지점의 관리 화면까지 모객 전 과정을 하나의 운영 체계로 만들었습니다.

## 핵심 내용

- **"계정 = 지점" 설계** — 고객 회원가입 없이 로그인 계정 자체를 지점 엔티티로 설계. 지점은 자기 데이터만, 본사는 전체를 보고, 미확인 건수를 사이드바 배지로 상시 노출
- **홈쇼핑 신청 시스템 & 다채널 확장** — 차수(round) 개념으로 회차별 데이터를 분리하고, 롯데홈쇼핑 채널 추가 시 기존 신청 데이터 **1,849건을 손실 없이 마이그레이션**. 채널·차수별 매장 라인업, 신청 가능 차수는 허용 리스트 한 줄로 제어
- **방문자 통계** — Django 미들웨어에서 요청을 가로채 IP+시간 단위 유니크 제약으로 중복을 제거한 집계 구축, 일별·30일 평균을 관리자 페이지에서 확인
- **권역 관리** — 지점을 권역으로 묶는 다대다 구조와 CRUD로 권역별 예약 관리·지역 마케팅 지원
- **엑셀 다운로드** — 관리자 권한별 필터링을 적용한 구매 데이터 내보내기
- **운영 DB 보호** — 로컬 개발은 환경 변수 하나로 빈 SQLite로 구동되도록 분리해 운영 데이터 오염 원천 차단
