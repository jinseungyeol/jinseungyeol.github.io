# 카페24 상품 이미지 WebP 자동 변환 앱

> 개발 · 참여 100% · NestJS, Next.js, PostgreSQL, Prisma, pg-boss, sharp, Cafe24 API

카페24 쇼핑몰의 상품 상세 본문 이미지를 WebP로 자동 변환해 **평균 84% 용량 절감**(실측 124.93MB → 20.03MB)을 달성한 풀스택 앱. 운영 중인 쇼핑몰에 적용하는 만큼 "원본을 훼손하지 않고 언제든 즉시 되돌릴 수 있는" 구조를 최우선으로 설계했습니다.

## 핵심 설계

- **무오염(clean-body)** — 본문 HTML은 영구 원본으로 두고, 스킨 head 최상단의 동기 스니펫이 상품 상세에서만 이미지 로드를 선점해 WebP로 교체. 원본 요청은 전송 전에 취소되어 전송량은 WebP뿐
- **롤백 = 상태 스위치** — 서빙 계층은 "변환 완료" 상태일 때만 WebP를 응답하고 그 외에는 원본으로 폴백. 롤백·재최적화가 본문 수정 0회, DB 상태 변경만으로 즉시 발효
- **장부(ledger) 설계** — DB에는 이미지가 아니라 "무엇을 최적화했고 어떻게 되돌리는지"만 기록. 위치의 장부와 내용의 장부를 분리하고 콘텐츠 해시로 연결해 중복 변환을 구조적으로 제거
- **URL 식별 + ETag 스윕** — 위치 기반 식별의 한계(이미지 삽입 시 밀림)를 URL 기준으로 전환하고, URL이 같은데 내용만 바뀌는 FTP 덮어쓰기는 CDN ETag(=콘텐츠 MD5, 실측 확인)를 지문으로 감지
- **운영 가드** — 매일 드리프트 스캔(신규 자동 압축·재매핑), 스니펫 3계층 가드(단일 소스 주입 → 배포 게이트 → 라이브 워치독), 운영 상태 대시보드
- **최소 권한** — 무오염 전환으로 쓰기 권한이 불필요해져 카페24 API 권한을 읽기 전용 단일 scope로 축소
