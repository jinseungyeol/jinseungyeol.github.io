보고서 자동발송 안내

이달에 생성된 보고서는 다음과 같은 상태에 따라 자동으로 발송됩니다. 아래 기준을 참고하여 자동발송 기능을 통해 업무 효율을 높여보세요!
자동발송 상태는 각 보고서의 상태에 따라 발송 여부가 결정됩니다. 대상이 아닌 보고서는 발송되지 않으며, 발송대상인 보고서만 자동 발송됩니다.

발송대상

발송대상 상태의 경우, 데이터 검증이 완료된 보고서로 자동발송 대상입니다.

대상아님

대상아님 상태의 경우, 아래 조건에 해당되면 자동발송 대상이 아닙니다.

  • 1. 거래처의 보고서 생성이 OFF로 설정된 경우

    해당 거래처의 보고서 생성이 비활성화되어 관련 데이터가 없거나 사용되지 않는 경우입니다.
    다시 보고서 생성을 원하시면 해당 화면에서 설정을 ON으로 활성화하면 보고서가 생성되고 검증 가능한 상태로 전환됩니다.
    보고서를 일괄 설정하고 싶다면 보고서 설정 메뉴에서 그룹을 추가한 후 일괄 거래처를 설정하는 것을 추천드립니다.
    (예상 부가세 보고서의 경우, 면세사업자는 보고서 생성에 제외됩니다.)

  • 2. 거래처의 보고서 데이터 검증이 실패한 경우

    검증 항목이 일부 누락되었거나 해당 거래처의 수임 동의 연결이 필요한 경우입니다.
    실패 상태인 거래처의 검증을 다시 진행하려면 수임 동의를 연결한 후 데이터 검증 버튼을 클릭하여 재시도해야 합니다.

  • 3. 거래처의 보고서 자동발송이 OFF로 설정된 경우

    해당 거래처의 보고서 자동발송이 비활성화되어 자동으로 발송되지 않는 경우입니다.
    다시 보고서 자동발송을 원하시면 해당 화면에서 설정을 ON으로 활성화하면 다음 달부터 보고서가 자동으로 발송됩니다.
    거래처별 보고서 설정은 보고서 관리 > 보고서 발송에서 언제든지 변경할 수 있습니다.

  • 4. 거래처의 보고서 자동발송 주기가 설정된 경우

    거래처를 그룹으로 지정하여 보고서별 자동 발송 주기를 매월이 아닌 분기 또는 반기로 설정한 경우입니다.
    설정된 주기에 따라 보고서 발송일이 결정되며, 자동 발송 주기 설정은 보고서 설정 > 보고서 그룹 설정에서 변경할 수 있습니다.
