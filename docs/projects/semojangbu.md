# 세모장부

> 퍼블리싱 · 2022.01 ~ 2022.12 · 신규 70% / 유지운영 30% · HTML, CSS, jQuery · 하이브리드 앱

소상공인 사장님을 위한 세무 컨설팅·장부 관리 하이브리드 앱의 퍼블리싱을 담당했습니다. 카드매출·홈택스 자료 연결, 장부·분석 등 복잡한 재무 데이터를 직관적으로 보여주는 것이 목표였고, **CSS 하드웨어 가속으로 화면 전환 프레임 속도를 2배 이상 끌어올리고** Mobiscroll.js 달력을 도입해 사용자 편의성을 높였습니다.

## 프로젝트 배경

네이티브 셸 위에 웹뷰를 올린 하이브리드 앱 특성상 저사양 기기에서 애니메이션·화면 전환이 버벅이는 문제가 있었고, 매출·세금 데이터라는 무거운 콘텐츠를 다루면서도 앱처럼 매끄러운 경험을 제공해야 했습니다. 신규 화면 퍼블리싱 70%와 기존 화면 유지운영 30%를 병행하며 1년간 참여했습니다.

## 주요 작업

| 영역 | 내용 |
|---|---|
| 화면 퍼블리싱 | 홈택스 연동(HT), 카드매출 관리(CA), 장부(C), 제휴/파트너(CP), 회원가입(SU), 고객지원(CS), 자산·분석 등 화면 ID 체계 기준 100여 개 화면 |
| 인터랙션 | jQuery 기반 탭·아코디언·바텀시트 팝업·토스트 등 동적 인터랙션 구현 |
| 라이브러리 적용 | Mobiscroll.js(달력·날짜 선택), Swiper(배너·슬라이드), Chart.js(매출 차트) 연동 마크업 |
| 유지운영 | 운영 중 화면의 수정·개선 및 다양한 기기·브라우저 환경 테스트 |

## 구현 포인트

- **CSS 하드웨어 가속 최적화** — 팝업·토스트·화면 전환 애니메이션을 `left/top` 변경 대신 `transform`(translate)과 `opacity` 조합으로 구현해 GPU 합성을 유도, 저사양 기기에서 프레임 속도를 2배 이상 개선
- **Mobiscroll.js 달력** — 조회 기간·날짜 선택이 많은 세무 앱 특성에 맞춰 달력·휠 피커 UI를 Mobiscroll.js로 통일해 네이티브에 가까운 조작감 제공
- **vw 단위 반응형 스타일** — 다양한 해상도의 모바일 기기에서 시안 비율이 그대로 유지되도록 주요 간격·크기를 vw 단위로 환산해 적용
- **화면 ID 체계 산출물 관리** — `proto_v2/` 아래 구분(HT·CA·C·CP 등)별 폴더와 화면 ID 파일명으로 정리하고, 전체 화면을 링크로 모은 워크리스트(index) 페이지를 만들어 기획·개발 파트와의 산출물 전달을 체계화
- **공통 스타일 분리** — `common.css`(공통 요소)·`main.css`(레이아웃)·`semo_style.css`(서비스 전용)·`partner-common.css`(제휴 화면)로 파일을 역할별로 분리해 신규 작업과 유지운영이 섞이지 않도록 관리

## 데모 구성

데모는 실제 산출물 구조 그대로, 워크리스트 페이지에서 구분·화면 ID별로 정리된 100여 개 마크업 화면을 새 창으로 열어 확인하는 방식입니다. 카드매출 연결, 홈택스 연동, 달력 선택 등 주요 플로우 화면을 직접 볼 수 있습니다.
