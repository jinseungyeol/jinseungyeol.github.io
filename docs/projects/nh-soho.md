# NH소상공인파트너

> 퍼블리싱 · 2022.01 ~ 2022.12 · 신규 40% / 유지운영 60% · HTML, CSS, jQuery

소상공인의 세무·경영 지원 자료를 제공하는 **NH소상공인파트너 하이브리드 앱**의 퍼블리싱 프로젝트입니다. 사용자가 지역별 세무 지원 정보와 보상 내역을 쉽게 확인·비교할 수 있도록 직관적인 UI를 구축하는 데 중점을 두었고, 신규 화면 구축과 유지운영을 1년간 병행했습니다. **SVN 버전 관리와 HTML 산출물 페이지 구축으로 업무 프로세스를 약 30% 단축**한 것이 대표 성과입니다.

## 프로젝트 배경

- 소상공인 대상 세무·경영 지원 서비스를 하이브리드 앱으로 제공하는 프로젝트로, 계좌·매출·지출·입금 등 금융 데이터를 다루는 다수의 화면이 필요했습니다.
- 개발사와 협업하는 구조여서, 퍼블리싱 산출물을 화면ID 단위로 체계적으로 전달·관리하는 프로세스가 중요했습니다.

## 주요 작업

- HTML, CSS, jQuery 기반의 UI 설계 및 구현 중심 퍼블리싱 (신규 40%, 유지운영 60%)
- 앱 내 데이터 표출과 사용자 인터랙션을 위한 동적 기능(탭, 아코디언, 차트, 스와이프 등) 마크업·스크립트 작업
- SVN 환경에서의 버전 관리 및 협업 프로세스 정립
- 화면ID 기반 HTML 산출물 목록 페이지를 별도 구축해 산출물 전달·검수 흐름 개선
- 다양한 디바이스 환경(하이브리드 앱 웹뷰)에서의 UI 테스트

## 구현 포인트 (데모 코드 기준)

- **화면ID 체계의 산출물 관리** — `proto_v2/` 아래 HT(홈), CR, CP, CN, POP(팝업) 등 업무 구분별 폴더에 약 175개의 화면을 `CHT001.html`처럼 화면ID 규칙으로 관리하고, `index.html` 산출물 목록(worklist)에서 구분·화면ID·비고 테이블로 전체 화면에 바로 접근할 수 있게 구성했습니다. 개발 측이 어떤 화면이 완료됐는지 즉시 확인할 수 있어 커뮤니케이션 비용이 줄었고, 이것이 업무 프로세스 30% 단축의 핵심이었습니다.
- **금융 데이터 시각화** — Chart.js로 매출·지출 등 수치 데이터를 그래프로 표현하고, Swiper로 카드형 콘텐츠 스와이프, Mobiscroll로 날짜·기간 선택 UI를 구현해 네이티브 앱에 가까운 조작감을 제공했습니다.
- **공통 리소스 분리** — `common.css`/`main.css` 공통 스타일과 화면 그룹별 스타일(`nh_style.css`, `semo_style.css`)을 분리하고, 아이콘은 SVG·폰트는 Pretendard 등 웹폰트로 통일해 175개 화면에서 일관된 UI를 유지했습니다.
- **접근성·구조 고려 마크업** — 목록은 `ul/li`, 화면 타이틀은 헤딩 구조로 마크업하고 상태는 `is-on` 같은 상태 클래스로 제어해 jQuery 이벤트 처리와 유지보수가 쉬운 구조를 만들었습니다.

## 데모 구성

| 경로 | 내용 |
|---|---|
| `index.html` | 화면ID 기반 HTML 산출물 목록(worklist) 페이지 |
| `proto_v2/` | 업무 구분(HT·CR·CP·CN·POP 등)별 화면 HTML 약 175개 |
| `css_v2/`, `js_v2/` | 공통 스타일과 jQuery 3.5, Chart.js 3.5, Swiper, Mobiscroll 등 라이브러리 |

## 배운 점

- 체계적인 버전 관리와 산출물 관리가 프로젝트 효율과 품질에 크게 기여함을 경험했습니다.
- 동적인 UI 구현과 유지보수를 고려한 퍼블리싱 역량이 프로젝트 완성도를 좌우한다는 점을 체감했습니다.
