# i-ONE 소상공인

> 퍼블리싱 · 2022.01 ~ 2023.12 · 앱(신규 40%, 유지운영 60%), 홈페이지(100%) · HTML, CSS, jQuery, GA4, AWS(S3, CloudFront, Route 53)

소상공인을 위한 금융·행정 지원 서비스의 하이브리드 앱과 홈페이지를 운영·고도화한 프로젝트입니다. 앱은 신규 기능과 유지보수를 병행해 퍼블리싱했고, 홈페이지는 퍼블리싱·프론트 개발·AWS 정적 배포까지 100% 담당했습니다. **정적 배포 파이프라인으로 클라이언트 수정 요청에 신속하게 대응**하고, **GA4로 사용자 흐름을 분석**하며 운영했습니다.

## 프로젝트 배경

사용자의 접근성과 편의성을 고려한 안정적이고 빠른 서비스 제공이 필요했습니다. 운영 기간 내내 클라이언트의 UI 개선 요청이 반복적으로 발생하는 프로젝트 특성상, 요청을 받아 빠르게 반영·배포할 수 있는 정적 배포 환경(S3 + CloudFront + Route 53)을 구성하는 것이 운영 효율의 핵심이었습니다.

## 주요 작업

| 영역 | 내용 |
|---|---|
| 하이브리드 앱 | 신규 기능 퍼블리싱과 기존 화면 유지보수 병행 (신규 40% + 유지운영 60%) |
| 홈페이지 | 디자이너와 협업한 전반적인 UI 퍼블리싱, jQuery 기반 동적 폼 처리·유효성 검증 |
| 배포·분석 | S3 정적 배포 + CloudFront 캐시 처리 + Route 53 도메인 관리, GA4 연동으로 사용자 분석·트래픽 관리 |

## 구현 포인트

- **정적 배포 기반의 빠른 운영 대응** — 수정 요청이 잦은 환경에서 S3 업로드와 CloudFront 캐시 처리만으로 반영이 끝나는 구조를 만들어, 다수의 클라이언트 요구사항을 짧은 주기로 반영. 퍼블리셔 역할을 개발·배포까지 확장한 계기가 된 프로젝트
- **GA4 사용자 흐름 분석** — 홈페이지에 GA4를 연동해 유입·이동 흐름과 트래픽을 분석하고 UI 개선 판단의 근거로 활용
- **공통 UI 스크립트 모듈화** — 하단 시트 팝업을 jQuery로 동적 생성하는 로직, 하나만 열리는 아코디언 메뉴, 금액 숫자 카운팅 애니메이션, 리스트 슬라이드 업/다운 등 반복 사용되는 인터랙션을 공통 `design.js`로 묶어 수백 개 화면에서 재사용
- **대규모 화면 관리 체계** — 260여 개 앱 화면을 모듈 코드(A·B·C·D·F·G·OB·OC·OG) 기준 폴더로 분류하고 화면ID 네이밍 규칙으로 관리해, 기획서와 퍼블리싱 산출물이 1:1로 대응되도록 정리

## 데모 구성

하이브리드 앱 퍼블리싱 산출물 원형을 그대로 담은 빌드입니다. 진입 페이지가 전체 화면ID 워크리스트로 구성되어 있어, 링크에 마우스를 올리면 iframe으로 즉시 미리보고, 클릭하면 쿼리스트링으로 경로를 전달받는 모바일 프레임 페이지에서 실제 앱 화면 비율로 확인할 수 있습니다.
