# 위멤버스 스케줄러

> 퍼블리싱 · 2024.01 ~ 2024.06 · 참여 100% · React, SCSS, TypeScript · 웹 앱

반복적인 4대 보험 신고·세무 자료 수집 업무를 자동화하는 스케줄러 웹 앱의 퍼블리싱을 전담했습니다. 월별 일정과 수집 처리 현황을 시각적으로 명확히 파악할 수 있는 UI를 목표로, React + TypeScript 기반 컴포넌트 퍼블리싱과 SCSS 스타일 구조 설계를 담당해 **디자인 시안과 1:1 매칭되는 화면 20여 종**을 구현했습니다.

## 프로젝트 배경

담당자가 매월 반복하던 4대 보험·세무 신고 업무를 자동화하는 서비스로, 사용자가 수집 일정을 등록하면 사업용 계좌·신용카드, 부가세, 원천세, 4대보험 고지내역 등을 정해진 주기에 자동 수집합니다. 복잡한 업무 흐름을 일정 중심으로 시각화해 사용자 부담을 줄이는 것이 핵심이었고, 디자이너와 UI 구조·흐름을 협의하며 캘린더·리스트·상태 뱃지 등을 컴포넌트 단위로 분리해 작업했습니다.

## 주요 작업

| 영역 | 내용 |
|---|---|
| 페이지 퍼블리싱 | 홈(일정 대시보드), 수집업무 추가, 조회 화면 10여 종(계좌·카드·부가세·원천세·4대보험), 로그인, 설정/예약설정, 자동수집 등 24개 페이지 |
| 공통 컴포넌트 | 헤더·LNB·업무 카드(TaskCard)·요일 선택(DaySelectBox)·날짜 선택(DatePicker, 연월/반기 선택)·로딩·토스트·툴팁·인증서 선택 팝업 등 13종 |
| 스타일 설계 | SCSS 전역 구조 설계(리셋 + 글로벌 + CSS Module 병행) 및 상태별 아이콘·진행률 시각 표현 |

## 구현 포인트

- **컴포넌트 단위 분리** — 반복 등장하는 업무 카드, 날짜/요일 선택, 상태 안내 요소를 props 기반 재사용 컴포넌트로 분리해 페이지 퍼블리싱 속도와 일관성을 확보
- **TypeScript 타입 기반 퍼블리싱** — 모든 컴포넌트의 props를 `type`/`interface`로 명시해 일정 정보·상태 아이콘 등 데이터와 UI 연결 시 오류를 컴파일 단계에서 차단
- **SCSS 3계층 구성** — `base.scss`(리셋·Pretendard 폰트 정의), `styles.scss`(전역 레이아웃·공통 컴포넌트 스타일, 중첩·`:has()` 셀렉터 활용), `intro.module.scss`(데모 목록 전용 CSS Module)로 역할을 분리해 스타일 충돌 없이 유지보수 효율 향상
- **라우터 기반 산출물 관리** — react-router-dom v7 중첩 라우트로 헤더·LNB 레이아웃을 공통화하고, 루트에 화면 ID·비고가 정리된 산출물 목록 페이지를 두어 기획·개발 파트가 화면 단위로 바로 확인 가능하도록 구성
- **빌드 환경** — Vite 6 + React 19 + TypeScript 5.7 기반, 빌드 시 라우트 프리렌더 스크립트를 거쳐 GitHub Pages에 정적 배포
